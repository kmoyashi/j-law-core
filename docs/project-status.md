# J-Law-Core プロジェクトステータスと免責

## 位置付け

J-Law-Core は、日本法令計算を整数演算と複数言語バインディングで表現する方法を検証するための **alpha / PoC / コンセプト実装**です。現時点の主目的は、設計、API、registry 構成、端数処理の再現方法を公開し、議論と改善を進めることにあります。

これは「選択した法令計算を実装してみる OSS」です。完成済みの業務基盤、法令準拠保証製品、専門家監修済みの計算エンジンであるとは扱わないでください。

## 何を保証しないか

- 計算結果の正確性、完全性、最新性
- 法改正、告示改正、通達、実務運用の変化への即時追随
- 個別事案への適合性
- 税務申告、契約、請求、源泉徴収、印紙税判断などの実務判断にそのまま使えること
- 将来バージョンとの後方互換性
- バグ、不備、解釈漏れが存在しないこと

コードの一部には AI 生成 / AI 補助による実装が含まれます。人手による全面レビュー、法務レビュー、税務レビュー、第三者監査は完了していません。

## 想定している使い方

- 実装パターンや API 設計の検討
- 端数処理や date-effective registry の試作
- 自前実装と比較するための参考コード
- クロス言語バインディングの PoC
- 研究・学習・検証用のサンプル

## 想定していない使い方

- 税務申告や届出の最終値を自動確定すること
- 契約書、見積書、請求書、精算処理の唯一の計算根拠にすること
- 顧客向け金額提示、報酬精算、法令適合性判定を無人で実行すること
- 専門家確認なしで本番業務に組み込むこと

## 現在の実装スコープ

現時点で PoC 実装があるのは次の 5 ドメインです。

- `consumption_tax`
- `real_estate`
- `income_tax`
- `stamp_tax`
- `withholding_tax`

ただし、これは各法令の全面実装を意味しません。各ドメインとも、現在コードと registry に入っている対象範囲だけを扱います。

## 品質シグナル

現状のリポジトリでは、少なくとも次を自動確認しています。

- `make ci` による format / clippy / Rust test
- GitHub Actions 上での Python / Node.js / Ruby / Go binding テスト
- `tests/fixtures/` を使ったクロス言語テストケースの共有
- JSON registry の妥当性チェック

これらは回帰検知や壊れた配布物の早期発見には有効ですが、法的正確性や業務利用可能性の認証ではありません。

## 実務で使うなら

1. まず一次資料と公式計算例を確認してください。
2. 自社・自案件の条件で追加テストを作ってください。
3. バージョンを固定し、アップデート時に差分検証してください。
4. `breakdown` や適用日付を確認し、前提条件を明示してください。
5. 最終判断は必ず有資格者または専門家が行ってください。

## ドキュメント上のルール

このリポジトリの公開ドキュメント、README、package metadata では、プロジェクトを **alpha / PoC** として扱ってください。「法的正確性を保証する」「本番利用可能」といった断定表現は避け、必要に応じてこの文書へリンクしてください。
